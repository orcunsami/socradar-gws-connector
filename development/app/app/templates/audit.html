What do these columns and codes mean?

Time — when it happened.   Actor — who/what did it: an operator's email, auto (the autonomous engine), or scheduler (the periodic Cloud Scheduler scan).   Action — what was attempted: scan, remediate:<action>, paired:<action> (a security follow-up such as revoking tokens after a reset), verify:<action> (re-reads the directory to confirm the effect landed), approval_*, settings, tenant_create.   Target — the affected user's email (— when there isn't one).

Detail on a scan row reads like totals={'botnet': 1, 'pii': 3} processed={...} in_scope=4 found=4:

  • totals — leaked records the feed returned, per source (botnet / pii / vip).
  • processed — how many were actually read (smaller than totals only when the feed was truncated).
  • in_scope — of the unique emails, how many fall inside your verified domains (the rest are skipped — a cross-domain lookup returns 403, not 404).
  • found — of the in-scope emails, how many actually exist in your Workspace directory → these become Flagged Users.
  • TRUNCATED=[…] — the feed had more pages than the page limit; raise FEED_MAX_PAGES or narrow the window so coverage isn't silently partial.
